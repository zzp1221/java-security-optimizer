# 03-Java分析引擎模块（JavaParser）

## 项目总体是做什么
项目是一款本地代码优化分析桌面应用，目标是在离线环境下给出稳定、可解释、可执行的代码改进建议。

## 本模块要干什么
- 扫描项目源码并建立 Java AST/符号上下文。
- 执行内置规则与自定义规则，输出标准化 Issue。
- 生成自动修复候选（FixCandidate），支持安全等级区分。
- 提供全量/增量分析模式，保障大仓库处理能力。

## 输入与输出
- 输入：`AnalyzeTaskRequest`（项目路径、规则集、模式、变更文件）。
- 输出：进度事件流 + `AnalyzeTaskResult`（issues、统计、耗时）。

## 第一批交付物
- 引擎主流程：index -> parse -> analyze -> report。
- 8 条高价值规则（空指针、资源关闭、字符串比较等）。
- 基础自动修复（字符串比较、循环拼接优化等）。

## 如何开始
- 初始化 `services/analysis-java`（Java 21 + Maven + JavaParser）。
- 先实现空引擎返回，再逐步接入解析与规则执行。
- 增加大文件保护参数：单文件大小阈值、并发度、超时。

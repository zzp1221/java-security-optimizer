# 08-插件扩展与 C++ 预留模块

## 项目总体是做什么
项目当前聚焦 Java 代码分析，但目标是演进为多语言离线代码优化平台，因此必须在第一版就固化插件化边界。

## 本模块要干什么
- 定义语言插件统一接口与能力声明。
- 保证 Java/C++ 输出同一 Issue/Fix 协议，前端无需改造展示层。
- 实现 Java 插件作为参考实现，C++ 提供占位插件与错误返回规范。
- 规范插件版本兼容策略，避免核心平台升级导致插件失效。

## 输入与输出
- 输入：插件元数据、分析请求、规则集配置。
- 输出：标准化进度事件、分析结果、能力协商结果。

## 第一批交付物
- `LanguagePlugin` 接口定义（meta/analyze/validateRulePack）。
- Java 插件实现（可运行）。
- C++ 占位插件（返回 NotImplemented + 能力声明）。

## 如何开始
- 在 `packages/plugin-sdk` 先冻结接口，再推动各模块实现。
- 在前端按 `language` 与 `supportsAutofix` 动态展示能力。
- 在任务提交流程中加入“插件可用性检查”，提前失败快返。

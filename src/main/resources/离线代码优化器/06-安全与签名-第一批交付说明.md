# 06 模块第一批交付说明（安全与签名）

## 交付范围
- 规则包本地导入场景的 `checksum + signature` 验证实现。
- 开发/生产密钥分离策略（同一 keyId 仅在所属环境生效）。
- 三类错误码体系：
  - `SIGNATURE_VERIFICATION_FAILED`
  - `ENGINE_VERSION_INCOMPATIBLE`
  - `PERMISSION_OUT_OF_BOUNDS`

## 验签流程（实现对应）
1. 导入入口先做目录授权校验（`RulePackSecurityContext#isPackagePathAuthorized`）。
2. 计算规则包文件 SHA-256，与 manifest.checksum 对比。
3. 校验 `engineVersionRange` 是否覆盖当前引擎版本。
4. 根据运行环境（DEV/PROD）解析受信任公钥：
   - 宽松模式：使用 manifest 内公钥（用于本地开发）。
   - 严格模式：从 `RulePackTrustedKeyStore` 按 `environment + keyId` 取公钥。
5. 对 canonical payload 执行签名验证：
   - 支持 `SHA256withRSA`
   - 支持 `SHA256withECDSA`
6. 校验通过后安装规则包；失败返回结构化错误码与原因。

## 开发/生产密钥分离策略
- 通过 `RulePackTrustedKeyStore` 维护环境级 key 集合。
- `ReleaseEnvironment.DEV` 仅信任 DEV keyId；`ReleaseEnvironment.PROD` 仅信任 PROD keyId。
- 如果 manifest 中 `keyId/algorithm/publicKey` 与环境信任库不一致，则验签失败并拒绝导入。
- 推荐策略：
  - 开发环境：允许宽松模式，便于联调。
  - 生产环境：必须启用严格模式，仅允许预置公钥。

## 错误码语义
- `SIGNATURE_VERIFICATION_FAILED`：签名值错误、keyId 不受信任、算法/公钥不匹配。
- `ENGINE_VERSION_INCOMPATIBLE`：规则包 engineVersionRange 与当前引擎版本不兼容。
- `PERMISSION_OUT_OF_BOUNDS`：规则包路径不在授权目录白名单内。

## 关键实现点
- `SignatureSpec` 增加 `keyId` 字段用于信任锚绑定。
- `RulePackValidator` 新增安全上下文重载接口，支持环境化验签。
- `RulePackImportService` 新增上下文重载接口，先做权限边界校验再进入验签链路。
- `RulePackValidationResult` 增加 `errorCode` 字段，支持前端/审计系统做稳定判定。

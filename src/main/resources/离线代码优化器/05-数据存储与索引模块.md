# 05-数据存储与索引模块（IndexedDB + 文件系统）

## 项目总体是做什么
项目要在本地高效处理代码分析任务，因此需要低延迟检索与可持续存储，支持大量 issue 与历史任务回溯。

## 本模块要干什么
- 使用 IndexedDB 保存结构化索引数据（任务、问题、规则包、工作区）。
- 使用文件系统保存大对象（完整报告、patch、快照）。
- 提供按项目、规则、文件、严重级别的高性能查询能力。
- 提供数据迁移与版本升级能力，避免客户端升级导致数据损坏。

## 输入与输出
- 输入：分析任务结果、规则包信息、用户筛选条件。
- 输出：分页问题列表、任务历史、报告路径、统计指标。

## 第一批交付物
- 表/仓库：`workspaces`、`tasks`、`issues_index`、`rulepacks`。
- 常用复合索引：`workspaceId+severity`、`workspaceId+ruleId`、`workspaceId+filePath`。
- 数据生命周期策略：历史任务归档、超大报告清理。

## 如何开始
- 在前端用 Dexie 建立 schema v1 与 migration v2 预留。
- 把 `issues` 主体存索引表，报告正文落本地文件路径。
- 为筛选面板提前设计查询 API，避免后期重构。

# 08 模块第一批交付说明（插件扩展与 C++ 预留）

## 交付范围
- 统一插件接口 `LanguagePlugin`，冻结 `meta/analyze/validateRulePack` 三个核心契约。
- Java 语言插件 `JavaLanguagePlugin`，直接接入现有 `JavaAnalysisEngine` 与 `RulePackValidator`，可实际运行分析与规则包校验。
- C++ 占位插件 `CppPlaceholderLanguagePlugin`，提供能力声明并按统一错误码返回 `NOT_IMPLEMENTED`。

## 插件接口契约
1. `meta()`：返回插件元信息（语言、版本、能力、是否支持自动修复、是否已实现）。
2. `analyze(request)`：接收统一分析请求，输出统一分析结果（Issue/Stats/Event）。
3. `validateRulePack(packageFile, manifest, securityContext)`：按统一规则包模型执行校验。

## 错误返回规范（首版）
- 错误类型：`PluginException`
- 错误码：`PluginErrorCode.NOT_IMPLEMENTED`
- 语义：插件已注册但功能尚未实现（典型场景：C++ 预留插件）

## Java 插件实现说明
- `JavaLanguagePlugin#analyze`：代理 `JavaAnalysisEngine#analyze`，保持现有 Java 分析链路不变。
- `JavaLanguagePlugin#validateRulePack`：代理 `RulePackValidator#validate`，复用当前验签与版本范围校验逻辑。
- 元信息声明 `supportsAutofix=true`，用于前端能力开关。

## C++ 预留实现说明
- 元信息声明：
  - `language=cpp`
  - `implemented=false`
  - `supportsAutofix=false`
- `analyze/validateRulePack` 当前均抛出 `PluginException(NOT_IMPLEMENTED, ...)`，用于任务入口快速失败。

## 测试覆盖
- `JavaLanguagePluginTest`
  - 验证 Java 插件可运行分析并产出结果。
  - 验证规则包校验链路可被插件调用并返回结构化错误码。
- `CppPlaceholderLanguagePluginTest`
  - 验证 C++ 插件能力声明。
  - 验证占位行为返回 `NOT_IMPLEMENTED`。

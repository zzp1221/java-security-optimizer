# 02-前端工作台模块（React + Monaco）

## 项目总体是做什么
项目用于在本地完成代码分析与优化建议生成，强调离线、安全、可扩展，第一阶段支持 Java，后续可接入 C++。

## 本模块要干什么
- 提供项目导入、任务发起、进度查看、结果检索的主交互。
- 使用 Monaco 展示源码与问题定位（行列级跳转）。
- 展示规则命中、严重级别、建议说明、修复前后差异。
- 提供规则管理界面（启停、分组、版本查看、规则包导入状态）。

## 输入与输出
- 输入：用户操作、桌面网关事件流、分析结果与规则数据。
- 输出：任务请求、规则操作请求、修复应用请求。

## 第一批交付物
- 页面：项目页、任务页、问题列表页、代码对比页。
- 状态模型：`workspace/task/issues/rulepacks`。
- 本地缓存：IndexedDB 索引读取与分页查询。

## 如何开始
- 初始化 `apps/web-ui`（Vite + React + TS）。
- 接入 Monaco 与路由，先用 mock 数据跑通页面。
- 与 Tauri IPC 对接，替换 mock 为真实任务流。

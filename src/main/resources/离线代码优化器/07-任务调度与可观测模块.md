# 07-任务调度与可观测模块

## 项目总体是做什么
项目需要在桌面端稳定执行长时间代码分析任务，并向用户持续反馈进度与结果质量，特别是大仓库场景。

## 本模块要干什么
- 管理任务生命周期：排队、执行、取消、失败重试、完成归档。
- 控制并发与资源：线程池、内存阈值、超时策略、降级策略。
- 产出可观测数据：耗时、规则命中数、失败原因、内存峰值。
- 提供统一追踪标识：`traceId/taskId` 跨前端、桌面层、Java 引擎贯通。

## 输入与输出
- 输入：任务提交请求、系统资源状态、用户取消动作。
- 输出：实时进度事件、任务状态、诊断日志、性能报表。

## 第一批交付物
- 任务状态机（Created/Queued/Running/Cancelled/Failed/Completed）。
- 进度事件协议（阶段、百分比、当前文件、附加消息）。
- 本地诊断面板（最近任务耗时、失败 TOP 原因）。

## 如何开始
- 在桌面层实现队列与取消令牌。
- 在 Java 引擎埋点四阶段耗时（index/parse/analyze/report）。
- 建立最小压测脚本，覆盖 100k/500k LOC 两档数据集。

# 04-规则中心与规则包模块

## 项目总体是做什么
本项目通过本地分析引擎识别代码问题并给出优化建议，支持团队将规则标准化并在离线环境共享与复用。

## 本模块要干什么
- 管理内置规则与用户自定义规则（启停、分组、优先级）。
- 支持规则包导入、版本管理、兼容性校验。
- 提供统一规则元数据（规则说明、严重级别、是否支持自动修复）。
- 输出规则命中统计，反哺规则治理与质量优化。

## 输入与输出
- 输入：规则包文件、规则启停操作、规则参数配置。
- 输出：可执行规则集、校验结果、版本与审计记录。

## 第一批交付物
- `RulePackManifest` 规范（packId/version/language/rules/signature）。
- 本地规则仓（已安装规则包列表 + 当前激活规则集）。
- 导入校验流程（checksum + signature + engineVersionRange）。

## 如何开始
- 在 `packages/rule-pack-sdk` 定义 manifest 与校验器。
- 在前端提供规则包导入和规则开关界面。
- 在 Java 引擎侧完成规则注册器与规则过滤执行逻辑。

# Java Security Optimizer - 使用文档

## 核心功能

- 静态规则分析：扫描 Java 代码中的安全与质量问题
- 问题分级与修复建议：输出严重级别、定位信息和可修复建议
- 任务调度：支持任务提交、状态跟踪和结果查询
- 规则包管理：支持规则包导入、校验与治理能力

## 基本使用流程

## 1. 提交分析任务

调用任务接口提交源码路径或上传包，系统返回任务 ID。

## 2. 查询任务进度

根据任务 ID 轮询状态，获取执行进度和阶段信息。

## 3. 获取分析结果

任务完成后查询问题列表，包含：

- 规则 ID
- 严重级别
- 文件与行号
- 说明与修复建议

## 4. 导入与启用规则包（可选）

- 导入签名后的规则包
- 校验兼容性与签名有效性
- 绑定项目规则策略并生效

## 开发建议

- 新规则开发：优先在 `analysis/rules` 中按单一职责实现
- 规则测试：为每条规则补充最小可复现测试
- 提交流程：采用 `feature/*` 分支，通过 PR 合入 `main`

## 版本与发布建议

- 标记里程碑版本：`vX.Y.Z`
- 发布前执行：`mvn test` + 前端 `npm run build`
- 发布说明包含：变更摘要、兼容性说明、回滚方案

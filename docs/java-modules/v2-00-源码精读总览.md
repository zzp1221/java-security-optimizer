# Java 模块学习（第二版）：源码精读路线

这份是第二版总览，目标是帮助你从“知道模块做什么”升级到“能读懂核心实现并改得动代码”。

## 第二版与第一版的区别

- 第一版：偏知识点地图，强调概念和模块职责。
- 第二版：偏源码调用链，强调“从入口到落地代码”的阅读路径。

## 文档目录（第二版）

1. `v2-01-analysis-源码精读.md`  
   重点：`JavaAnalysisEngine` 的主流程、并发执行、缓存与降级机制。

2. `v2-02-task-storage-源码精读.md`  
   重点：`TaskSchedulerService` 调度循环、状态机约束、重试取消语义。

3. `v2-03-plugin-rulepack-security-源码精读.md`  
   重点：插件健康检查、规则包导入链路、验签与审计闭环。

4. `v2-04-autofix-governance-源码精读.md`  
   重点：自动修复应用链路、质量门禁与回滚、规则治理策略。

## 建议阅读顺序

1. 先读 `v2-02`（任务调度），建立全局执行主线。  
2. 再读 `v2-01`（分析引擎），理解核心计算。  
3. 再读 `v2-03`（插件与安全），理解扩展与可信边界。  
4. 最后读 `v2-04`（自动修复与治理），理解工程闭环。

## 每篇文档的统一结构

- 模块入口：从哪个类/方法开始看
- 调用链：按“谁调谁”展开
- 关键决策：为什么这样设计
- 常见改造点：如果要新增功能，应该改哪里
- 练习任务：可直接动手验证

# Java 模块学习总览（按项目拆分）

你希望“更详细 + 分模块”，所以我把后端 Java 部分拆成 5 个专题文档。  
建议按下面顺序阅读，每一篇都对应真实源码与测试。

## 文档目录

1. `01-analysis-分析引擎详解.md`  
   主题：JavaParser AST、规则模型、并发执行、缓存、降级策略、性能与正确性。

2. `02-task-storage-任务调度与状态管理.md`  
   主题：任务队列、优先级、超时/重试/取消、状态机、线程安全、诊断指标。

3. `03-plugin-rulepack-security-插件与规则包安全.md`  
   主题：插件架构、版本兼容、规则包导入、签名验签、权限边界、安全审计。

4. `04-autofix-自动修复链路详解.md`  
   主题：Fix 计划生成、Patch 合并、冲突检测、质量门禁、回滚机制。

5. `05-governance-规则治理与策略配置.md`  
   主题：模板化规则集、参数治理、误报反馈、治理仪表盘、仓储抽象。

## 怎么用这组文档学习

- 第 1 轮（理解）：先看每篇的“模块职责”和“主流程图（文字版）”。
- 第 2 轮（落地）：跟着“关键类索引”在 IDE 里跳转阅读源码。
- 第 3 轮（巩固）：完成每篇最后的练习题，并补 1 个测试用例。
- 第 4 轮（进阶）：对比测试类，理解“为什么这样设计”而不只是“怎么实现”。

## 统一阅读方法（每篇都适用）

- 先看输入输出：模块接收什么，产出什么。
- 再看状态变化：哪些字段在变化，在哪里保证一致性。
- 再看失败路径：超时、异常、非法输入、回滚。
- 最后看扩展点：如果我要新增能力，最小改动在哪里。

## 你可以先从哪里开始

- 如果你更关注并发：先读 `02-task-storage-任务调度与状态管理.md`。
- 如果你更关注静态分析：先读 `01-analysis-分析引擎详解.md`。
- 如果你更关注安全：先读 `03-plugin-rulepack-security-插件与规则包安全.md`。
- 如果你更关注“可自动改代码”：先读 `04-autofix-自动修复链路详解.md`。

# Java Security Optimizer 30 天学习计划（第三版配套）

目标：30 天内完成“读懂 -> 能改 -> 能测 -> 能讲”。

---

## 第 1 周：搭框架（模块认知）

### Day 1-2

- 阅读 `v2-00`、`v3-00`，建立全局地图。
- 输出 1 张你自己的模块关系图。

### Day 3-4

- 学 `v3-02-task-storage-方法级清单.md` 前半部分。
- 在 IDE 跟 `submit -> workerLoop -> runManagedTask`。

### Day 5-7

- 学 `v3-01-analysis-方法级清单.md` 前半部分。
- 跟 `analyzeWithMetrics` 全流程并写流程笔记。

周目标：
- 能口述任务从 API 到分析执行的主链路。

---

## 第 2 周：啃核心（并发 + 规则）

### Day 8-10

- 深读 `parseFiles`、`runRules`、缓存逻辑。
- 记录 3 个你认为最可能出性能问题的点。

### Day 11-12

- 精读 3 条规则实现（建议 NPE、String、Resource）。
- 自己新增 1 条简单规则（如 println 检测）。

### Day 13-14

- 看 `JavaAnalysisEngineTest`，补 1 个你新规则的测试。

周目标：
- 能独立新增规则并跑通测试。

---

## 第 3 周：安全与扩展（插件 + 规则包）

### Day 15-17

- 学 `v3-03-plugin-rulepack-security-方法级清单.md`。
- 画出导入链路（controller -> importService -> validator -> audit）。

### Day 18-19

- 深读 `RulePackValidator`（checksum/compatibility/signature）。
- 写一份“验签失败场景清单”。

### Day 20-21

- 学 `PluginManagerService` 版本兼容逻辑。
- 补 1 个版本范围边界测试。

周目标：
- 能解释规则包为什么“可验证且可追踪”。

---

## 第 4 周：自动修复与治理（闭环）

### Day 22-24

- 学 `v3-04-autofix-governance-方法级清单.md` 的 AutoFix 部分。
- 跟 `applyBatch` 各分支，画时序图。

### Day 25-26

- 学 Governance 部分：规则集计算、参数清洗、指标累计。
- 解释“模板 + 覆盖”的策略模型。

### Day 27-28

- 做一个小改造：给 dashboard 增加新指标（例如规则单位收益）。
- 补 1-2 个测试。

### Day 29-30

- 整体复盘：写 1 份“架构讲解稿”（15 分钟）。
- 自测：从零描述一次任务执行、一次规则包导入、一次自动修复回滚。

周目标：
- 形成“发现问题 -> 修复问题 -> 治理策略”的完整闭环认知。

---

## 每日固定动作（建议）

- 10 分钟：看前一天笔记，复盘关键方法。  
- 30 分钟：按文档打卡方法源码。  
- 20 分钟：做一个小实验（改参数、加日志、跑测试）。  
- 10 分钟：记录“今天一个新认知 + 一个疑问”。  

---

## 验收标准（30天后）

- 能独立讲清 4 条核心链路：
  1. 任务调度链路
  2. 分析执行链路
  3. 规则包导入安全链路
  4. 自动修复与回滚链路
- 能独立完成 2 个小功能改造并补测试。
- 能指出当前实现至少 5 个可优化点并给出方案。
